# SessionGraph Web UI Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build a dark-mode developer-tool web dashboard for browsing, searching, and exploring reasoning chains stored in Supabase.

**Architecture:** Next.js 15 App Router with server components for initial data loads, client components for interactive features (search, filters). Browser talks directly to Supabase via `@supabase/ssr` вЂ” no custom API layer. Auth via Supabase email/password. Middleware refreshes sessions on every request. Zustand for client-side state management.

**Tech Stack:** Next.js 15, TypeScript, Tailwind CSS v4, shadcn/ui (new-york style), Zustand, `@supabase/ssr`, Lucide icons, Vitest + Playwright, Vercel hosting.
